I have already used the term "web-page" 3 times in this web-page. Now 5. And before we even try to understand what does it mean to program a web-page with JavaScript, we first need to understand and agree on what the Web is.

The Internet is the technology that enables multiple computers to talk to each other while being geographically distributed inter-connected over physical network. The Web is another layer of abstraction on top of the Internet, being a set of documents inter-connected via links, i.e. a web of documents.

Simply put, Some computers on the internet have names, like "google.com" and "wikipedia.org". When you enter the computer name in the address bar of your browser, the browser talks to that computer and asks it to send back the document named "index.html" in the root directory of the computer. If you want another document, you can specify the name of the desired document using a slash (/) like this: "js.bagrat.io/another-document.html"

For example, if we take the computer named "js.bagrat.io", typing in the address bar "js.bagrat.io" or "js.bagrat.io/index.html" will return the same document, which is "index.html" on computer "js.bagrat.io". You might have noticed the ".html" suffix multiple times already. Don't worry, we will talk about it soon.

Now this "index.html" is just a single document. I authored and put this document on the "js.bagrat.io" computer. Since I control this computer, I have also put another document to demonstrate the web to you. That other document is next to the "index.html" document in the root directory, named "the-web.html". So if you remember the slash (/) you have just learnt about you might have already guessed that the address of that document in the Web is "js.bagrat.io/the-web.html".
